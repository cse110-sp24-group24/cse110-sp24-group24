**2. User Story Process & Deadline**

- Sam explained the purpose of user stories: they should all be written from the perspective of the same defined user — a new or up-and-coming CS student — rather than spanning a variety of different user types. The variety should come from the features themselves, not the users.
- Nishant had already fed the MVP MD file into an AI tool and generated 10 user stories, which Sam praised and asked him to share with the team.
- The goal is for every team member to independently write 10 user stories. With the full team contributing, the group expects a large pool (roughly 90-100) with duplicates, which will then be filtered and sorted — Sam noted he would use Claude or Gemini to assist with that.
- Sam emphasized this brainstorming step is the most important creative part of the process, since AI cannot generate these ideas on behalf of the team.
- **Deadline:** User stories due by Wednesday, to be shared as part of the standup.
- Sam mentioned he may create a shared document so everyone can paste their stories in one place for easier review.
- Friday's meeting will be used to filter through all the user stories and plan how to begin implementing them.
- Coding is not expected to start until at least the weekend, pending any direction from class on Tuesday.
